## D-06｜折扣券折抵金額偏少

- **模組**：結帳計價
- **嚴重度**：Medium（AI 原標 Minor，本報告上調）
- **PRD**：R-4.7（計算基準）、R-2.3（四捨五入）
- **對應測試**：`tests/test_coupon_discount.py`

### 重現步驟

1. 登入，清空購物車。
2. 加入「機械式鍵盤」×1（商品小計 NT$3,180）。
3. 前往結帳，套用 PCT15（全站 85 折券）。
4. 觀察「優惠券折抵」金額。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-4.7） | 折抵 = 商品小計 × 折扣率 = 3,180 × 15% = **477**（不以扣滿額折扣後金額為基準）。 |
| **實際** | 折抵顯示 **453**（短少 24 元，使用者多付），似以「小計 − 滿額折扣」再乘 15%。 |

### 根本原因分析

折扣券計算基準錯誤：以「商品小計扣掉滿額折扣後的金額」計算，違反 R-4.7「以商品小計為計算基準」。

計價引擎有套用券，但摘要列「優惠券折抵」未綁定正確折抵欄位（或渲染成 0），造成金額細項與應付不一致。

## D-15｜通知未讀徽章未穩定顯示於導覽列

- **模組**：導覽列／通知中心
- **嚴重度**：Medium
- **PRD**：R-1.5
- **對應測試**：`tests/test_nav_extras.py::test_通知中心徽章反映未讀則數且為零時不顯示數字`

### 重現步驟

1. 登入並完成一筆下單（未讀通知增加）。
2. 觀察導覽列「通知中心」是否出現未讀數字徽章。
3. 進入通知中心確認「未讀 N 則」N≥1。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-1.5） | 未讀 >0 時導覽顯示徽章數字；為 0 時不顯示；超過 99 顯示「99+」。 |
| **實際** | 通知中心頁內「未讀 N 則」正確，但導覽徽章表現不穩定：部分跑測可見「通知中心 N」，部分僅「通知中心」無數字。 |

### 根本原因分析

未讀數未正確同步到導覽徽章元件，或徽章僅在特定路由渲染。

## D-16｜送出訂單未顯示「處理中…」／未停用

- **模組**：結帳
- **嚴重度**：Medium
- **PRD**：R-12.8
- **對應測試**：`tests/test_checkout_extras.py::test_送出訂單時按鈕顯示處理中並防重複下單`

### 重現步驟

1. 購物車有商品，進入結帳並填妥收件資訊。
2. 點「送出訂單」，觀察按鈕文案／停用狀態（可於慢速網路下更明顯）。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-12.8） | 點擊後按鈕立即停用並顯示「處理中…」，直到系統回應；同一次操作只建立一張訂單。 |
| **實際** | 可能直接進入完成頁，未觀察到「處理中…」或按鈕停用回饋。 |

### 根本原因分析

前端未在送出期間切換 loading 狀態，或狀態切換過快無法感知；需確認是否仍有防重複提交保護。

## D-17｜待審核缺少「撤銷退貨申請」按鈕

- **模組**：退貨
- **嚴重度**：**High**
- **PRD**：R-7.12、R-14.9
- **對應測試**：`tests/test_return_extras.py::test_待審核可撤銷退貨申請`

### 重現步驟

1. 完成訂單 → 出貨 → 收貨 → 申請退貨（原因 ≥5 字）。
2. 於「退貨中／待審核」訂單詳情觀察操作區。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-7.12） | 提供「撤銷退貨申請」按鈕；確認後回到已完成／無退貨。 |
| **實際** | 僅見「賣家審核（Demo）」，**沒有**撤銷按鈕。 |

### 根本原因分析

退貨待審核操作區未實作撤銷流程（UI／API 皆缺）。

## D-18｜已使用券訂單編號缺少 `MM-` 前綴

- **模組**：我的優惠券
- **嚴重度**：Low
- **PRD**：R-4.12、R-17.7、R-6.9
- **對應測試**：`tests/test_coupon_lifecycle.py::test_下單後使用的優惠券變為已使用並記錄完整訂單編號`

### 重現步驟

1. 結帳選用一張可用券並下單，記下完整訂單編號（含 `MM-`）。
2. 開啟「我的優惠券」→「已使用」頁籤，查看該券卡片。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-6.9／R-17.7） | 顯示完整訂單編號，例如 `MM-20260718-0021`。 |
| **實際** | 顯示 `訂單編號 20260718-0021`（缺少 `MM-` 前綴）。 |

### 根本原因分析

渲染已使用券時只輸出日期流水號片段，未套用 R-6.9 完整格式。

## D-19｜免運券下單後仍留在「可使用」

- **模組**：優惠券／結帳
- **嚴重度**：**High**
- **PRD**：R-4.12
- **對應測試**：`tests/test_coupon_lifecycle.py::test_下單後使用的優惠券變為已使用並記錄完整訂單編號`

### 重現步驟

1. 購物車加入「純棉素色 T 恤」×1，進入結帳。
2. 選取「免運券」（運費應變 NT$0），送出訂單。
3. 開啟「我的優惠券」→「已使用」與「可使用」頁籤。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-4.12） | 該張免運券進入「已使用」，並記錄對應訂單編號。 |
| **實際** | 結帳運費已為 NT$0（券有套用），但免運券**仍在「可使用」**，未進入「已使用」。 |

### 根本原因分析

結帳計價有套用免運，但券狀態機未將 FREESHIP 標記為已使用（或寫回失敗），導致同一張券可被重複選用。
