## D-03｜加入購物車後導覽列徽章未即時更新

- **模組**：商品列表／導覽列徽章
- **嚴重度**：Medium
- **PRD**：R-1.4
- **對應測試**：`tests/test_cart_badge.py`

### 重現步驟

1. 登入，確認購物車為空（徽章不顯示或為 0）。
2. 於商品列表點「加入購物車」。
3. **立即**觀察導覽列購物車徽章數字（不重新整理）。
4. 重新整理或進入購物車頁後再觀察。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-1.4） | 徽章數字為購物車內所有商品數量總和；加入後應即時反映。 |
| **實際** | 加入後徽章不更新；需重新整理或切換到購物車頁才正確。 |

### 根本原因分析

商品列表的加入購物車動作未觸發導覽列徽章即時重算；購物車頁有重算，兩頁行為不一致。

## D-06｜折扣券折抵金額偏少

- **模組**：結帳計價
- **嚴重度**：Medium（AI 原標 Minor，本報告上調）
- **PRD**：R-4.7（計算基準）、R-2.3（四捨五入）
- **對應測試**：`tests/test_coupon_discount.py`

### 重現步驟

1. 登入，清空購物車。
2. 加入「機械式鍵盤」×1（商品小計 NT$3,180）。
3. 前往結帳，套用 PCT15（全站 85 折券）。
4. 觀察「優惠券折抵」金額。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-4.7） | 折抵 = 商品小計 × 折扣率 = 3,180 × 15% = **477**（不以扣滿額折扣後金額為基準）。 |
| **實際** | 折抵顯示 **453**（短少 24 元，使用者多付），似以「小計 − 滿額折扣」再乘 15%。 |

### 根本原因分析

折扣券計算基準錯誤：以「商品小計扣掉滿額折扣後的金額」計算，違反 R-4.7「以商品小計為計算基準」。

## D-10｜取消訂單後庫存未回補

- **模組**：訂單取消／商品庫存
- **嚴重度**：**High**（取消可用時）／Blocked（取消未實作時）
- **PRD**：R-3.7、R-6.5
- **對應測試**：`tests/test_stock_effects.py::test_取消訂單後庫存應回補`
- **驗證狀態**：受阻（cand2 取消 API／UI 404；且 mid 斷言先撞上 D-09）

### 重現步驟

1. 記錄 `stock_before` → 下單 1 件 → 斷言 `stock_mid == stock_before − 1`。
2. 於訂單詳情取消訂單。
3. 斷言庫存回補至 `stock_before`。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-6.5、R-3.7） | 取消後庫存加回購買數量。 |
| **實際** | 環境無取消功能；即便有取消，在 D-09 未扣庫存時無法單獨證回補。 |

### 根本原因分析

## D-12｜下單確認通知內容未對應該筆訂單

- **模組**：通知中心
- **嚴重度**：High
- **PRD**：R-8.2、R-15.7（內文組成見 R-15.2）
- **對應測試**：`tests/test_notifications.py`

### 重現步驟

1. 登入，清空購物車。
2. 以數量 3 加入「手沖咖啡濾杯」，收件人填「通知對帳買家」，完成下單，記下訂單編號。
3. 進入「通知中心」，找到標題「訂單 {訂單編號} 已成立」那一則，比對內文。

### 預期行為 vs 實際行為

| | 內容 |
|--|------|
| **預期**（R-8.2） | 標題含本筆訂單編號；內文為本筆的下單時間、商品明細「手沖咖啡濾杯 × 3」、應付金額、收件人「通知對帳買家」。不同訂單的通知內容不得相同。 |
| **實際** | 標題訂單編號正確，但內文固定為內建範例資料（如「手沖咖啡濾杯 × 2」「應付金額 NT$1,020」「收件人 測試收件人 1」「下單時間 2026-07-01 10:15」），與剛下的訂單無關。多品項訂單同樣如此。 |

### 根本原因分析

產生下單確認通知時，標題帶入了正確 `orderId`，但 body 似複製／快取了內建訂單（A.4／A.5）的固定文案，未依本筆訂單的品項、數量、應付金額與收件人組裝（違反 R-8.2「每一則對應自己那一筆訂單」）。
